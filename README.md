# 🔐 Oasis InfoByte Summer Internship Program (OIBSIP)
## Track: Cyber Security

This repository contains my submissions for the **Oasis InfoByte Summer Internship Program (OIBSIP)** under the **Cyber Security** track.

The Cyber Security track consists of **10 tasks**, and candidates are required to complete **at least 4 tasks** to successfully finish the internship. Tasks are divided into **Beginner (Tasks 1–6)**, **Intermediate (Tasks 7–8)**, and **Advanced (Tasks 9–10)** levels. Tasks can be selected freely across tiers based on skill level.

---

## 📌 Deliverables Overview

Tasks in this track are divided into two main deliverable types:

**Practical Tasks (1, 2, 3, 7, 8, 9, 10)**  
These require:
- A GitHub repository containing scripts, command output, and screenshots
- A screen-recorded demo video showing task execution or results

**Research / Report Tasks (4, 5, 6, 10)**  
These require:
- A well-structured Markdown (`.md`) report committed to GitHub  
- No demo video is required for pure research tasks

---

## 🟢 Beginner Level Tasks (1–6)

**Task 1 — Basic Network Scanning with Nmap**  
Perform network scanning on a local system or VM using Nmap to identify open ports, running services, OS details, and assess potential security risks.

**Task 2 — Basic Firewall Configuration with UFW**  
Configure a host-based firewall using UFW on a Linux system by allowing and denying specific traffic, verifying enforcement, and automating rules with a script.

**Task 3 — SQL Injection on DVWA (Low Security)**  
Demonstrate a classic SQL Injection vulnerability using DVWA in a controlled local environment, document payloads, exposed data, and mitigation techniques.

**Task 4 — Research Report: Common Network Security Threats**  
Write a structured research report covering major network threats, real-world examples, impacts, and mitigation strategies.

**Task 5 — Research Report: Social Engineering Attacks**  
Research phishing, pretexting, baiting, and related attacks with case studies and prevention recommendations.

**Task 6 — Research Report: Importance of Patch Management**  
Explain patch management, its role in vulnerability mitigation, real-world breaches caused by unpatched systems, and best practices.

---

## 🟡 Intermediate Level Tasks (7–8)

**Task 7 — Vulnerability Scanning with Nikto**  
Use Nikto to scan a local test web server, analyse findings, categorise vulnerabilities, and propose remediation steps.

**Task 8 — Capture Network Traffic with Wireshark**  
Capture and analyse live network traffic, apply protocol filters, examine unencrypted data, and explain security implications.

---

## 🔴 Advanced Level Tasks (9–10)

**Task 9 — Advanced SQL Injection Exploitation**  
Extend SQL injection testing to more advanced techniques, enumerate database structures, optionally compare manual exploitation with sqlmap, and provide remediation guidance.

**Task 10 — Full Network Security Assessment Report**  
Conduct a multi-phase security assessment of a local test network using tools like Nmap, Wireshark, and Nikto, and produce a professional assessment report.

## 📁 Repository Structure

Each task is stored in its own folder using the following naming convention:

CyberSecurity-Task[Number]-[ShortDescription]/

Each task folder typically contains:
- A task-specific `README.md`
- Scripts, logs, or report files
- A `screenshots/` directory for evidence

## ⚠️ Ethical Use Statement

All security testing, scanning, exploitation, firewall configuration, and traffic capture activities in this repository were performed **only on systems I own or explicitly control**, such as local virtual machines and isolated test environments.  
No external, third-party, or production systems were targeted.
